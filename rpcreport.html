  5. Briefly explain the protocol you use over TCP to request function calls and return the results.
  6. Our protocol relies on null termination never naturally occurring for the data that will be sent over the network. Because we were concerned about the network byte ordering, we convert all data to their string form before sending it, along with a null character to signify its end. This is true for the function names (signifying the start of the RPC), and the final messages as well (DONE or BAD).

    We made this assumption because all valid types (string, int, float, struct (composed of any combination of the first three), array (one of the previous ones), void) that will be sent over the network guarantee no null characters will ever be present in the data.
